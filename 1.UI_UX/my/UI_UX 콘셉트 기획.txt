UI/UX 콘셉트 기획

UI(UserInterface) : 사용자가 직접 보는 화면 디자인
(버튼, 아이콘, 색상, 레이아웃 등)
UX(User eXperience) : 사용자가 서비스를 이용하면서 경험하는 모든 과정

왜 중요한가?
사용자의 문제를 해결할 수 있도록 서비스를 설계하는 과정이다.

기획 순서
1. 아이디에이션(ideation)
- 아이디어를 창의적으로 떠올리는 과정
- 브레인 스토밍, 마인드맵 등 여러가지 기법이 있다.

브레인스토밍 : 질보다는 양에 초점을 맞춰서 토의하는 방식
마인드맵 : 하나의 주제에서 꼬리에 꼬리를 물고 내려오는 방식

2. 프로토타입 검증
- 서비스의 초기 버전
- 사용자가 프로토타입을 사용해본 후 피드백을 받아 수정

3. 최종 UI/UX 콘셉트를 확정
- 사용자 니즈와 비즈니스 목표를 반영해 콘셉트 결정
--------------------------------------------------------------------------------
페르소나(Persona) 추출
- 서비스의 대표적인 사용자 유형을 가상의 인물로 설정
- 가상의 인물로 '우선 순위 사용자'를 세분화 하고 구체화 한다.
- 사용자 조사(User Research)
	- 인터뷰, 설문조사, 시장조사를 활용하기
- 페르소나 프로필 작성
	- 이름, 연령, 직업, 관심사, 목표 정리하기
- 페르소나 기반 서비스 기획
	- 예를 들어, 20대 직장인 페르소나라면 바쁜 일정 속에서도 빠르게 이용할 수 있는 UI가 필요할 것이다.
-----------------------------------------------------------------------------------------
사용자 시나리오
- 사용자가 서비스를 이용하는 과정을 시나리오 형태로 정리하는 단계

1. 사용자 목표 정의
- 사용자가 원하는 것은 무엇인지 정리

2. 사용자의 행동 플로우 작성
- 예를 들어 "사용자가 로그인 후 상품을 검색하고 결제하는 과정" 정리

3. 각 단계에서의 사용 경험 개선
- 불편한 요소가 없는지 확인하고 UI 개선점을 정리

----------------------------------------------------------------------------------------
사용자 스토리(User Story) 작성
- 사용자가 원하는 기능을 한 문장으로 정리하는 방법
- 사용자 역할 : 온라인 쇼핑몰 사용자
- 필요한 기능 : 상품을 장바구니에 담을 수 있음
- 이유 : 한 번에 여러 개의 상품을 결제하고 싶어서

-----------------------------------------------------------------------------------------
스토리보드(Storyboard) 작성
- 사용자 시나리오를 시각적으로 표현
- UI 설계를 구체적으로 할 수 있고, 개발자와 디자이너가 쉽게 이해할 수 있다.
-----------------------------------------------------------------------------------------
모바일 쇼핑 앱 사용과 관련된 리서치 결과
- 목표 : 사용자가 모바일 쇼핑 앱에서 상품을 검색하고 장바구니에 담는 과정에서 겪는 불편함을 조사하여 UI/UX를 개선한다.
